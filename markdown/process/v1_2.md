## 背景与原则
- 当前 FastAPI 后端使用 prisma-client-py，会在启动时连接 Query Engine，导致 EngineConnectionError 不能启动。
- 我决定在 V1.1 之前把 Prisma/事务/落库迁移到 Next.js（Node）层，Python 后端只做推理。
- 需要保持现有 `/debug` 接口语义尽量兼容（输入字段保持，输出字段保持），以减少前端改动成本。
- 代码必须加入详细中文注释（我后端不熟）。

## 必做改动范围
请修改/删除以下后端文件中的 DB 相关内容：
- `app/db.py`：删除 Prisma Client 相关代码（或保留空壳但不再被引用）
- `app/main.py`：移除 startup/shutdown 中 connect_db/disconnect_db；删除/禁用 `/sessions` 与 `/sessions/{session_id}`（这些将迁移到 Next）
- `app/services/debug_service.py`：移除所有落库逻辑；保留 prompt 构造、LLM 调用、JSON 解析与 schema 校验
